# ClaudeCoach 前端功能说明

## 1. 前端定位

ClaudeCoach 的“前端”不是传统浏览器网站，而是一个运行在 VS Code 侧边栏里的 Webview。

前端技术栈非常直接：

- `src/sidebar/webview/index.html`：界面骨架
- `src/sidebar/webview/style.css`：样式
- `src/sidebar/webview/main.js`：状态管理、事件绑定、消息通信
- `src/sidebar/SidebarProvider.ts`：Webview 与扩展宿主之间的桥接层

可以把它理解成：

```text
VS Code 宿主进程
  -> 注入 Webview HTML/CSS/JS
  -> 接收前端 postMessage
  -> 调用课程 / 资料 / AI / 诊断模块
  -> 把结果再 postMessage 回前端
```

前端本身不直接做 AI 调用，也不直接读写课程文件。它只负责：

- 展示当前状态
- 收集用户操作
- 通过消息把动作发给宿主层
- 接收结果后重新渲染页面

## 2. 前端总结构

前端被拆成 5 个一级标签页：

- `学习`
- `对话`
- `资料库`
- `设置`
- `日志`

它们共用一套中心状态对象 `state`，主要字段包括：

- `courses`
- `materials`
- `preferences`
- `diagnosis`
- `selectedSubject`
- `selectedCourseMaterialId`
- `currentCourseMaterialPreview`
- `chatGroundingMode`
- `chatMessages`
- `activeTaskKeys`
- `resolvedAIConfig`

核心初始化伪代码如下：

```pseudo
Webview.init():
  vscode = acquireVsCodeApi()
  state = restoreSavedState()
  cacheDomElements()
  bindTabEvents()
  bindCourseEvents()
  bindMaterialEvents()
  bindChatEvents()
  bindSettingsEvents()
  bindWindowMessageListener()

  renderCourseDropdown()
  renderSelectedCourse()
  renderMaterials()
  renderDiagnosis(null)
  renderChatContext()
  renderResolvedAIConfig(null)

  postMessage("getCourses")
  postMessage("getMaterials")
  postMessage("getPreferences")
  postMessage("getDataDir")
  postMessage("getResolvedAIConfig")
```

## 3. 学习页模块

### 3.1 课程选择与创建模块

职责：

- 显示已有课程下拉菜单
- 切换当前课程
- 进入“创建新课程”模式
- 为预设学科或自定义学科生成课程大纲

模块输入：

- 课程列表 `courses`
- 当前选择 `selectedSubject`
- 用户输入的学科名

模块输出：

- `generateCourse`
- 课程切换
- 创建模式开关

伪代码：

```pseudo
renderCourseDropdown():
  if currentSubject exists in courses:
    label = currentSubject
  else if draftSubject exists:
    label = "准备创建: " + draftSubject
  else:
    label = "请选择课程"

onGenerateCourseClick():
  subject = getDraftSubject()
  if subject is empty:
    showWarn("请先填写课程名称")
  else:
    postMessage({ type: "generateCourse", subject })
```

### 3.2 课程树模块

职责：

- 展示课程标题、主题、课时
- 打开已有讲义
- 生成或重生成讲义
- 生成或打开练习
- 重置某一课时
- 标记某一课时已完成

课程树是当前项目里最核心的前端展示区。每个课时会显示：

- 标题
- 进度状态点 `not-started / in-progress / completed`
- 讲义按钮
- 更多操作菜单

模块化描述：

```text
模块名：CourseTree
输入：CourseOutline
状态：selectedSubject
输出：
  - openLessonContent
  - openOrGenerateLesson
  - openOrGenerateExercises
  - resetLessonProgress
  - markLessonCompleted
```

伪代码：

```pseudo
renderSelectedCourse(course):
  for topic in course.topics:
    drawTopic(topic)
    for lesson in topic.lessons:
      drawLessonRow(lesson, lesson.status)

onLessonRowClick():
  postMessage("openLessonContent")

onLessonButtonClick():
  postMessage("openOrGenerateLesson")

onExerciseMenuClick():
  if action == "exercise":
    postMessage("openOrGenerateExercises")
  if action == "reset":
    postMessage("resetLessonProgress")
  if action == "complete":
    postMessage("markLessonCompleted")
```

### 3.3 课程资料模块

职责：

- 展示当前课程下的资料列表
- 显示资料索引状态
- 在学习页内预览当前资料
- 删除资料
- 选择当前资料作为聊天上下文来源

资料状态有 3 种：

- `pending`
- `extracted`
- `indexed`

如果当前资料存在预览内容：

- Markdown 直接简化渲染
- TXT / PDF 提取文本以纯文本块展示

伪代码：

```pseudo
renderCourseMaterials():
  materials = allMaterials.filter(subject == selectedSubject)
  if materials empty:
    show "暂无资料"
  else:
    renderList(materials)

onMaterialClick(materialId):
  postMessage({ type: "previewMaterial", materialId })

onDeleteMaterial(materialId):
  postMessage({ type: "requestDeleteMaterial", materialId })
```

### 3.4 学习诊断模块

职责：

- 展示最近一次诊断结果
- 手动触发诊断运行

UI 展示的重点内容：

- 掌握度进度条
- 推荐聚焦点
- 总体策略
- 下一步建议

伪代码：

```pseudo
requestDiagnosis(run):
  if no selectedSubject:
    show "请先选择课程"
    return
  postMessage({ type: "getDiagnosis", subject: selectedSubject, run })
```

## 4. 对话页模块

### 4.1 聊天模式切换

聊天页支持 3 种上下文模式：

- `course`：当前课程
- `material`：所选资料
- `general`：普通问答

这 3 种模式直接决定发给后端的上下文强度。

模块化描述：

```text
模块名：ChatContextSwitcher
输入：selectedSubject, selectedCourseMaterialId
状态：chatGroundingMode
输出：chat 请求中的 mode / subject / materialId
```

伪代码：

```pseudo
onChatSend():
  text = chatInput.value.trim()
  appendChat("user", text)

  postMessage({
    type: "chat",
    message: text,
    subject: mode == "general" ? undefined : selectedSubject,
    mode: chatGroundingMode,
    materialId: mode == "material" ? selectedCourseMaterialId : undefined
  })
```

### 4.2 AI 回复展示模块

职责：

- 保存聊天历史到 Webview state
- 把 assistant 返回内容做轻量 Markdown 渲染
- 自动滚动到底部

当前渲染器是项目内置的轻量版本，不是完整 Markdown 引擎。它主要支持：

- 标题
- 列表
- 普通段落

### 4.3 基于资料重构课程大纲入口

聊天页额外提供一个快捷动作：

- `基于资料重构当前大纲`

它不是普通聊天，而是直接触发后端重构流程：

```pseudo
onRebuildOutlineClick():
  postMessage({
    type: "rebuildCourseOutline",
    subject: selectedSubject,
    materialId: selectedCourseMaterialId
  })
```

## 5. 资料库页模块

职责：

- 按课程维度汇总所有资料
- 从全局资料库跳回某门课程
- 直接预览资料
- 删除资料

它与学习页中的“课程资料”不同：

- 学习页只看当前课程
- 资料库页看全局所有课程资料

跳转逻辑伪代码：

```pseudo
onLibraryMaterialClick(item):
  state.selectedSubject = item.subject
  activateTab("learn")
  postMessage({ type: "previewMaterial", materialId: item.id })
```

## 6. 设置页模块

### 6.1 AI 配置中心展示模块

设置页内有一个很大的 `AI CONFIG CENTER` 区域。

从实际代码看，目前已接通的前端能力只有：

- 展示当前生效配置
- 展示 provider / model / Base URL / context window / max tokens / history budget
- 展示 warning pills
- 折叠 / 展开 AI 配置中心

也就是说，当前前端实际完成的是“AI 配置总览面板”。

### 6.2 已预留但前端未接线的 AI 配置表单

HTML 中已经预留了下面这些 UI：

- Profile 列表
- 新建 Profile
- 编辑 Provider / Base URL / Token / Model
- 项目级覆盖
- 从 `.claude` / `.codex` / 配置包导入
- 导出与测试按钮

但 `main.js` 里目前没有对应事件绑定，也没有处理：

- `aiProfiles`
- `aiImportResult`
- `aiTestResult`

所以现状应描述为：

```text
AI 配置后端能力已存在；
AI 配置前端界面骨架已存在；
但 Webview 交互目前只接通了“当前生效配置展示”，还没有完整接通“表单编辑流”。
```

### 6.3 学习偏好模块

职责：

- 设置整体难度
- 设置练习难度配比
- 设置每次练习题量
- 设置学习节奏
- 设置讲义语言与代码注释语言

伪代码：

```pseudo
onSavePreferences():
  preferences = collectFromForm()
  postMessage({ type: "savePreferences", preferences })
```

### 6.4 数据目录模块

职责：

- 显示当前数据目录
- 调用宿主在文件管理器中打开目录

## 7. 日志与任务模块

前端维护两个轻量可视化区域：

- `log-list`：日志流
- `task-list`：正在运行的任务列表

任务与按钮禁用是联动的：

```pseudo
onTaskStart(key):
  activeTaskKeys.add(key)
  disableMainButtons()

onTaskEnd(key):
  activeTaskKeys.remove(key)
  if no task running:
    enableMainButtons()
```

这让 UI 不会在同一时间重复触发同一类长任务。

## 8. 前后端消息协议

前端常用请求如下：

| 前端动作 | 发送消息 | 作用 |
| --- | --- | --- |
| 初始化课程 | `getCourses` | 获取课程树 |
| 生成课程 | `generateCourse` | 新建课程大纲 |
| 打开/生成讲义 | `openOrGenerateLesson` | 课时讲义 |
| 打开/生成练习 | `openOrGenerateExercises` | 课时练习 |
| 获取诊断 | `getDiagnosis` | 读取或运行诊断 |
| 导入资料 | `importMaterial` | 导入 PDF/TXT/MD |
| 预览资料 | `previewMaterial` | 学习页或资料库预览 |
| 对话 | `chat` | AI 问答 / 讲义修改 |
| 保存偏好 | `savePreferences` | 写入学习偏好 |
| 获取 AI 生效配置 | `getResolvedAIConfig` | 展示当前 AI 配置 |
| 打开数据目录 | `openDataDir` | 打开本地目录 |

宿主回传的核心消息如下：

- `courses`
- `courseGenerated`
- `materials`
- `materialPreview`
- `preferences`
- `diagnosis`
- `chatResponse`
- `taskStart`
- `taskEnd`
- `resolvedAIConfig`
- `log`
- `error`
- `dataDir`

## 9. 前端真实功能边界

从当前代码出发，前端已经真实完成的功能有：

- 课程创建、选择、重命名、移除
- 讲义生成 / 打开
- 练习生成 / 打开
- 课时状态展示、重置、标记完成
- 课程资料导入、预览、删除
- 学习诊断触发与展示
- 三模式聊天
- 基于资料重构课程大纲
- 学习偏好保存
- 当前 AI 生效配置展示
- 日志和任务态展示

从当前代码出发，前端尚未完整接线的功能有：

- AI 配置中心表单编辑流
- AI Profile 列表渲染与切换
- AI 导入结果 / 测试结果展示
- 通过侧边栏直接提交练习答案
- 通过侧边栏触发自动扫描批改
- 扩展命令面板发来的 `triggerGenerateCourse` 等触发消息

## 10. 一句话总结

这个前端不是“通用网页 UI”，而是一个围绕课程树、资料预览、AI 问答和本地学习文件工作流构建的 VS Code 学习控制台。它已经把主学习链路接通了，但 AI 配置中心和部分命令入口仍处在“界面已铺好、交互未完全接线”的阶段。
